# 인공지능 에이전트 아키텍처 및 프레임워크가 등장하고 있습니다

> 저자: Cobus Greyling (Chief Evangelist @ Kore.ai)
> 출처: Medium, 2024-09-16

## 요약

AI 에이전트를 구성하는 6계층 아키텍처 피라미드가 수렴되고 있으며, LAM(대규모 액션 모델)이 그 기반이 된다.

## 핵심 포인트

- 6계층 피라미드 (하→상): 모델 오케스트레이션 → LAM → Vision → Agentic Apps → Function Calling → Tools/HITL
- LAM(Large Action Model): 텍스트 이해를 넘어 실제 행동을 수행하는 모델
	- 클릭·입력 등 UI 조작 가능, LLM과 구별됨
- Human-in-the-Loop이 최상위 계층
	- 자동화할수록 인간의 검증이 더 중요해짐

## 인사이트

에이전트를 "LLM + 도구"로만 보는 시각은 좁다. LAM처럼 행동 자체를 학습한 모델이 기반이 되어야 진정한 에이전트가 가능하다. HITL을 계층 맨 위에 둔 것이 인상적 — 자동화와 통제는 반대가 아니라 세트다.

---

## 핵심 주장

차세대 AI 에이전트를 정의할 **근본적인 아키텍처 원칙들이 수렴**되기 시작하고 있다. 이 아키텍처는 단순한 고급 모델을 훨씬 뛰어넘는다.

## AI 에이전트 아키텍처 계층 구조

```
┌──────────────────────────────────────────────┐
│  The Role of Tools: Pipelines & HITL         │  ← 최상위
├──────────────────────────────────────────────┤
│  Function Calling & Structured Output        │
├──────────────────────────────────────────────┤
│  Agentic Applications & Agentic Discovery    │
│  Within Digital Environments                 │
├──────────────────────────────────────────────┤
│  Vision-Enabled Language Models              │
│  For Digital Exploration                     │
├──────────────────────────────────────────────┤
│  Large Action Models (LAMs)                  │
├──────────────────────────────────────────────┤
│  Model Orchestration Of Leveraging           │
│  Large & Small Language Models               │  ← 기반
└──────────────────────────────────────────────┘
```

## AI 에이전트란?

> "AI 에이전트는 하나 이상의 대규모 언어 모델(LLM) 또는 기초 모델(FM)을 핵심 기반으로 사용하여 자율적으로 작동할 수 있는 프로그램이다."

- 질의를 분해하고, 일련의 이벤트를 계획·생성하여 복잡한 문제 해결
- 인간의 추론과 유사한 사고 과정으로 매우 모호한 질문 처리 가능
- 프로그램, API, 웹 검색 등 다양한 도구에 접근 가능
- 자율적으로 또는 반자율적으로 작업 수행 (인간 개입도 도구로 활용)

주목할 만한 연구: Apple의 **Ferret-UI**, **WebVoyager**, Microsoft의 **OmniParser**, Apple의 **ToolSandBox**

## 1. 모델 오케스트레이션: 대형 + 소형 언어 모델

### LLM의 한계

- **환각(Hallucination)**: 일관성 있고 그럴듯해 보이지만 사실과 다른 답변 생성
- **지식 만료**: 학습 데이터의 만료일로 인해 최신 정보 제공 불가
- 파인튜닝으로 해소하려면 상당한 데이터 준비·비용·테스트 필요

### RAG + SLM 조합

```
SLM (소규모 언어 모델)
    + RAG (검색 증강 생성)
    = 대규모 LLM 수준의 견고성
```

- **SLM**: 개발·테스트에 이상적, 로컬/오프라인 모드 실행 가능
- **RAG**: SLM의 심층 지식 부족을 보완 → 더 큰 시스템과 유사한 성능
- 에이전트가 리소스 집약적 대형 모델 대신 소형 모델로 특정 기능 처리 가능

## 2. 대규모 액션 모델 (LAMs)

> "LLM이 자연어 처리를 혁신했던 것처럼, **대규모 액션 모델(LAM)은 AI 에이전트가 환경과 상호작용하는 방식을 혁신**할 것으로 기대된다."

| | LLM | LAM |
|--|-----|-----|
| 강점 | 비정형 콘텐츠 이해·생성 | 언어를 구조화되고 실행 가능한 행동으로 변환 |
| 출력 | 텍스트 | 구조화된 행동 |
| 역할 | 언어 처리 | 환경과의 물리적 상호작용 |

- Salesforce AI 연구소: 소규모 액션 모델 포함 LAM 오픈소스 공개
- **함수 호출(Function Calling)** 능력이 LAM 등장의 핵심 동인
- AI가 정보 검색, 작업 예약, 계산 수행 등 실제 작업 실행 가능

## 3. 디지털 탐색을 위한 시각 기반 언어 모델

에이전트가 디지털 환경을 탐색·상호작용하게 되면서 **시각 기능과 언어 모델의 통합**이 핵심으로 부상.

**주요 프로젝트**:
- **Apple Ferret-UI**: 모바일 UI 화면에서 위젯 분류, 아이콘 인식, OCR 수행. 상세 설명과 인식 대화를 통해 목표 지향적 행동 제안
- **WebVoyager**: 웹 탐색 AI 에이전트

**에이전트의 시각 기반 탐색 능력**:
- UI 요소 인식 (OCR)
- 경계 상자 매핑
- 텍스트 해석 후 요소와 상호작용
- 시각적 피드백 제공

> *예시: 새 환경에 애플리케이션을 설치하는 AI 에이전트 — 텍스트 지침 읽기 + UI 요소 인식 + 경계 상자 매핑 + 텍스트 해석 + 시각적 피드백 제공*

## 4. 함수 호출 및 구조화된 출력

전통적 LLM: 비정형 입력 → 비정형 텍스트 출력
→ 이제 **함수 호출**을 통해 구조화되고 실행 가능한 출력으로 전환

> "LLM은 비정형 콘텐츠를 이해하고 생성하는 데 유용하지만, LAM은 언어를 구조화되고 실행 가능한 행동으로 변환하여 이러한 격차를 해소하도록 설계되었다."

**함수 호출의 이점**:
- 회의 예약, 요청 전송, API 호출 등 특정 기능을 효율적인 토큰 사용으로 수행
- 비정형 응답 처리 오버헤드 감소
- 시스템 간 상호 작용 원활화

**중요**: OpenAI API로 함수를 호출할 때 **모델이 함수를 직접 실행하지 않음** — 호출할 함수와 인자만 반환하고, 실제 실행은 애플리케이션 코드가 담당.

## 5. 에이전트형 애플리케이션 및 에이전트형 발견

- **Agentic Discovery (Self-Discover)**: 에이전트가 적절한 추론 모듈을 스스로 선택·조합
  - SELECT → ADAPT → IMPLEMENT 단계
  - 문제 유형에 맞는 추론 구조를 동적으로 결성

## 6. 도구의 역할: 파이프라인 및 인간 참여형 프로세스

> "도구는 AI 에이전트가 데이터를 가져오거나, 계산을 수행하거나, 작업을 실행하는 등 세상과 상호작용하는 메커니즘이다."

- 도구 = 입력값을 한 단계에서 다른 단계로 전달하고 변환하는 **파이프라인**
- **Human-in-the-Loop (인간 개입형) 도구**: 사람이 중요한 순간에 개입하여 에이전트 행동 안내·검증
  - 의료, 금융 등 절대적 정확성이 필요한 고위험 환경에 특히 유용
- 도구는 에이전트의 기능을 확장할 뿐 아니라 **다양한 시스템을 연결하는 접착제** 역할

## AI 에이전트의 미래

> "대규모 액션 모델, 모델 오케스트레이션, 비전 기반 언어 모델, 함수 호출, 그리고 도구가 결합되어 인간의 문제 해결 및 사고 과정과 유사한 방식으로 행동할 수 있는 AI를 위한 토대를 마련하고 있다."

- 이러한 구성 요소들이 통합되면서 **더 정교하고 자율적인 에이전트** 가능
- 에이전트는 디지털 생태계의 진정한 일원으로 기능

## 시사점

- AI 에이전트 아키텍처는 단일 기술이 아닌 **6개 계층의 융합**으로 정의됨
- LLM → LAM으로의 진화: 언어 이해에서 실제 행동 실행으로
- SLM + RAG 조합이 비용 효율적 에이전트 구현의 현실적 대안
- 시각 기능 통합이 디지털 환경 탐색 에이전트의 핵심 요건
- Human-in-the-Loop는 완전 자율화의 위험을 관리하는 실용적 접근

# 딥러닝 라이브러리 비교

## 개념 요약

딥러닝 프로젝트 아키텍처 설계 시 라이브러리 선택 기준 비교표.

### 주요 딥러닝 프레임워크

| 라이브러리 | 개발자 | 주요 특징 | 장점 | 단점 | 적합 분야 |
|-----------|-------|---------|------|------|---------|
| **PyTorch** | Meta AI | Dynamic Graph, Eager 실행 | 디버깅 용이, 연구 유연성, 커뮤니티 방대 | 프로덕션 설정 복잡 | 연구, 프로토타입, NLP, CV |
| **TensorFlow** | Google Brain | Static/Dynamic Graph, TPU 지원 | 프로덕션 성숙도, TF Serving, 생태계 풍부 | API 복잡, 무거움 | 대규모 서비스, 엔터프라이즈 |
| **Keras** | Google (François Chollet) | TF 고수준 API, 직관적 설계 | 빠른 프로토타이핑, 코드 간결 | 커스터마이징 한계 | 입문, 프로토타입, 교육 |
| **JAX** | Google DeepMind | Functional, XLA JIT 컴파일 | 속도 최적, TPU 최적화, 자동미분 | 생태계 미성숙, 배포 도구 부족 | 연구, 수치 계산, TPU 활용 |

### 보조 라이브러리

| 라이브러리 | 개발자 | 주요 특징 | 장점 | 단점 | 적합 분야 |
|-----------|-------|---------|------|------|---------|
| **NumPy** | NumFOCUS | N차원 배열 연산, CPU 전용 | 단순, 범용, 의존성 최소 | GPU 미지원, 딥러닝 직접 구현 불가 | 수치 계산, 데이터 전처리 |
| **Pandas** | PyData | DataFrame 기반 데이터 조작 | 데이터 조작 최강, 직관적 API | 대용량 메모리 한계, ML 직접 불가 | 데이터 분석, 전처리 파이프라인 |
| **Scikit-learn** | INRIA | 전통 ML 알고리즘 모음 | 전통 ML 완성도, 파이프라인 지원 | 딥러닝 미지원, GPU 미지원 | 전통 ML, 피처 엔지니어링 |
| **MXNet** | Apache/AWS | Hybrid Graph, AWS 최적화 | AWS 통합, 경량 | 커뮤니티 축소, 개발 둔화 | AWS 기반 서비스 |

## 수식

$\text{모델 학습 속도} \propto \frac{\text{메모리 대역폭} \times \text{병렬 처리 수}}{\text{모델 파라미터 수}}$

---

## 직관적 설명
```
연구/실험 → PyTorch (유연성 최우선)
빠른 프로토타입 → Keras (코드 간결)
대규모 프로덕션 → TensorFlow (성숙한 생태계)
TPU/고성능 연산 → JAX (속도 최적화)
데이터 전처리 → Pandas + NumPy
전통 ML → Scikit-learn
```

---

## 구현 (코드)
```python
# 프레임워크 선택 가이드
def select_framework(purpose: str, scale: str) -> str:
    if purpose == "research":
        return "PyTorch"
    elif purpose == "prototype":
        return "Keras"
    elif scale == "large" and purpose == "production":
        return "TensorFlow"
    elif purpose == "tpu_optimization":
        return "JAX"
    elif purpose == "data_processing":
        return "Pandas + NumPy"
    elif purpose == "traditional_ml":
        return "Scikit-learn"
```

---

> [!tip] 아키텍처 선택 기준
> - **연구/프로토타입**: PyTorch (유연성), Keras (빠른 구현)
> - **중소규모 서비스**: PyTorch + TorchServe
> - **대규모 배포**: TensorFlow + TF Serving + Kubernetes
> - **엣지 디바이스**: TFLite (TF), ExecuTorch (PyTorch)
